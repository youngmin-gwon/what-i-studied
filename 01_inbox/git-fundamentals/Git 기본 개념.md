# Git 기본 개념

## Git이란?
Git은 코드의 변화를 추적하는 도구입니다. 마치 컴퓨터 게임의 세이브 포인트처럼, 작업한 내용을 저장하고 언제든 되돌아갈 수 있게 해줍니다.

## 핵심 개념들

### 리포지토리 (Repository)
- Git이 관리하는 프로젝트 폴더
- 모든 파일과 변경 기록이 저장되는 곳
- 줄여서 "repo"라고 부름

### 커밋 (Commit)
- 특정 시점의 코드 상태를 저장하는 것
- 각 커밋은 고유한 ID(해시)를 가짐
- 메시지와 함께 저장하여 무엇을 바꿨는지 기록

### 브랜치 (Branch)
- 코드의 다른 버전을 동시에 작업할 수 있게 해주는 기능
- main/master 브랜치가 기본 브랜치
- 새 기능이나 실험을 위해 새 브랜치 생성 가능

### 워킹 디렉토리 (Working Directory)
- 현재 작업 중인 파일들이 있는 곳
- Git이 추적하는 실제 프로젝트 폴더

### 스테이징 영역 (Staging Area)
- 다음 커밋에 포함될 파일들을 임시로 모아두는 곳
- `git add` 명령으로 파일을 스테이징 영역에 추가

### HEAD
- 현재 위치를 가리키는 포인터
- 대부분의 경우 현재 브랜치의 최신 커밋을 가리킴

## Git의 세 가지 상태
1. **Modified (수정됨)**: 파일을 변경했지만 아직 Git이 추적하지 않음
2. **Staged (스테이지됨)**: 변경된 파일이 다음 커밋에 포함되도록 표시됨
3. **Committed (커밋됨)**: 변경사항이 Git 데이터베이스에 안전하게 저장됨
