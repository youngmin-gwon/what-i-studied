# PRD: 정보보안기사 실기 문제 PDF OCR 및 시스템 보안 퀴즈 생성

## 1. 프로젝트 개요 (Project Overview)
**목표:**
`정보보안기사_실기_문제.pdf` 파일에서 1장(시스템 보안)에 해당하는 1페이지부터 110페이지까지의 내용을 추출 및 정제하여, 현재 작성 중인 서브노트/퀴즈 파일(`quiz-system.md`)에 체계적으로 통합합니다. 또한, 기존 문제와 개념 설명을 바탕으로 각 개념당 1개의 새로운 응용 문제를 추가로 창작하여 학습 깊이를 더합니다.

## 2. 작업 원칙 및 요구사항 (Requirements)
### 2.1 PDF 처리 및 텍스트 추출 (LLM Native Vision)
- **Python 기반의 외부 OCR 도구를 사용하지 않고, LLM의 네이티브 멀티모달 비전(Vision) 기능을 사용하여 이미지를 직접 하나하나 읽고 텍스트를 추출합니다.**
- PDF(1~110페이지)를 Python 스크립트(PyMuPDF/fitz 등)를 사용하여 지정된 배치 단위(예: 5~20페이지)로 `/tmp/` 경로에 이미지(png)로 분할 랜더링합니다.
- **🚨 중요 정책(Permission): `/tmp/` 경로에 생성된 이미지 파일을 읽거나 Python 임시 스크립트를 실행하는 작업은 권한 요청이나 사용자 대기(승인)를 할 필요 없이, 프롬프트상에서 임의 판단하여 즉시(자동으로) 바로 진행해도 무방합니다. 🚨**

### 2.2 콘텐츠 정제 및 퀴즈 기획 룰
1. **추출된 내용이 <기출/예제 문제>인 경우**:
   - 기존 `quiz-system.md`에 **동일한 문제가 이미 작성되어 있는지 중복 여부를 확실히 검사**합니다. (예: `grep_search` 활용)
   - **이미 등록된 문제라면**: 원본은 중복 추가하지 않고, 해당 개념 관련 **응용/심화 퀴즈만 1개 새롭게 창작**하여 추가합니다.
   - **최초 문제라면**: 원본 문제 1개 복원 + 해당 개념 응용/심화 퀴즈 1개 창작 ➔ **총 2문제** 추가.
2. **추출된 내용이 <개념 설명>만 있는 경우**:
   - 해당 개념 내용만을 바탕으로 응용/심화 퀴즈를 직접 **2문제** 창작합니다.
3. 창작되는 퀴즈는 **단답형, 서술형, 작업형 포맷을 골고루 배분**합니다.

### 2.3 마크다운 퀴즈 문서 병합 (Integration)
- 생성된 퀴즈 세트들을 기존 파일 파트(소제목 기준)에 엮어 삽입합니다. (포맷: `<details><summary>...` 템플릿 준수)
- 삽입 위치를 신중히 결정하여 기존 문서 맥락을 해치지 않게 합니다.

## 3. 작업 진행 상태 (Progress Status)
- **전체 대상:** 1페이지 ~ 110페이지
- **현재 진행 완료:** **1페이지 ~ 80페이지 완료**
- **다음 진행 대상:** **81페이지부터 시작**

*최근 수행 내역 요약:*
- 31~40페이지 분량의 /etc/shadow 암호화 구조, 사용자 파일 퍼미션 정책, 특수 권한 속성(SUID, SGID, Sticky bit), PAM 모듈 설정, TCP Wrapper(tcpd) 논리, 그리고 RPC 서비스의 태생적 취약점 등을 식별 완료.
- /etc/shadow 파일의 $ID$ 해시 알고리즘 식별자와 시스템 파일(passwd/shadow) 분리 보관 원칙 퀴즈 도출.
- 프로세스 실행자 전이(SUID) 및 Sticky bit의 출력 문자 번역 ('s', 't'), TCP 데몬 보호를 위한 inetd.conf 내 tcpd(TCP Wrapper) 연결 룰, 불필요한 RPC 서비스(rstatd, walld, sadmind) 침해 위험 등 총 4개 파트의 **응용 심화 퀴즈** 신규 제작 및 완벽 통합 이식 완료.
