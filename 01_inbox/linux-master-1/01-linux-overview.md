# 리눅스 개요 및 역사

## 1. 리눅스의 탄생과 발전

### 1.1 유닉스(UNIX)의 역사
- **1969년**: AT&T 벨 연구소에서 켄 톰슨과 데니스 리치가 UNIX 개발
- **1970년대**: C 언어로 재작성되어 이식성 향상
- **1980년대**: 상업용 유닉스 시스템 등장 (Sun Solaris, HP-UX, IBM AIX)
- **BSD 계열**: 버클리 대학에서 개발한 BSD(Berkeley Software Distribution)

### 1.2 리눅스의 시작
- **1991년**: 리누스 토르발스(Linus Torvalds)가 헬싱키 대학에서 개발 시작
- **초기 버전**: Minix를 기반으로 한 개인 프로젝트
- **0.01 버전**: 1991년 9월 17일 최초 공개
- **GPL 라이선스**: 자유 소프트웨어 재단의 GNU GPL 채택

### 1.3 주요 발전 단계
```
1991년 - Linux 0.01 (10,239 lines of code)
1994년 - Linux 1.0 (176,250 lines)
1996년 - Linux 2.0 (SMP 지원)
2001년 - Linux 2.4 (USB, 블루투스 지원)
2003년 - Linux 2.6 (향상된 스케줄러)
2011년 - Linux 3.0
2015년 - Linux 4.0
2019년 - Linux 5.0 (현재 안정 버전)
```

## 2. GNU 프로젝트와 오픈소스

### 2.1 GNU 프로젝트
- **1983년**: 리처드 스톨먼(Richard Stallman)이 시작
- **목표**: 완전히 자유로운 유닉스 호환 운영체제 개발
- **GNU 도구**: gcc, glibc, bash, coreutils 등
- **GNU/Linux**: 리눅스 커널 + GNU 도구의 조합

### 2.2 자유 소프트웨어와 오픈소스
**자유 소프트웨어의 4가지 자유**:
1. 프로그램을 어떤 목적으로든 실행할 자유
2. 프로그램의 작동 원리를 연구하고 변경할 자유
3. 복사본을 재배포할 자유
4. 개선된 버전을 배포할 자유

**주요 라이선스**:
- **GPL (General Public License)**: 카피레프트, 파생물도 GPL
- **LGPL**: 라이브러리용, 상업 소프트웨어와 연결 가능
- **BSD/MIT**: 허용적 라이선스, 재배포 자유
- **Apache**: 특허권 명시적 허가

### 2.3 오픈소스 개발 모델
- **분산 개발**: 전 세계 개발자들의 협업
- **버전 관리**: Git을 통한 소스 코드 관리
- **커뮤니티**: 메일링 리스트, 포럼, IRC
- **기여 방식**: 패치 제출, 코드 리뷰, 버그 리포트

## 3. 리눅스 배포판

### 3.1 주요 배포판 계열

**Debian 계열**:
- **Debian**: 안정성 중시, 커뮤니티 기반
- **Ubuntu**: 사용자 친화적, 6개월 주기 릴리스
- **Linux Mint**: Ubuntu 기반, 데스크톱 최적화
- **Kali Linux**: 보안 테스팅 전문

**Red Hat 계열**:
- **RHEL (Red Hat Enterprise Linux)**: 상업용, 기업 지원
- **CentOS**: RHEL 클론, 무료 (2021년 CentOS Stream으로 전환)
- **Fedora**: 최신 기술 테스트베드
- **Rocky Linux/AlmaLinux**: CentOS 대체

**SUSE 계열**:
- **openSUSE**: 커뮤니티 버전
- **SLES (SUSE Linux Enterprise Server)**: 상업용

**독립 배포판**:
- **Arch Linux**: 롤링 릴리스, 미니멀리즘
- **Gentoo**: 소스 기반 컴파일
- **Slackware**: 가장 오래된 배포판 중 하나

### 3.2 배포판 선택 기준
- **용도**: 서버, 데스크톱, 임베디드
- **안정성**: 장기 지원(LTS) vs 최신 기능
- **패키지 관리**: APT, YUM/DNF, Pacman
- **커뮤니티**: 문서화, 지원 수준
- **라이선스**: 완전 자유 vs 독점 드라이버 포함

## 4. 리눅스의 특징

### 4.1 기술적 특징
- **다중 사용자**: 여러 사용자가 동시에 시스템 사용
- **다중 작업**: 여러 프로세스 동시 실행
- **이식성**: 다양한 하드웨어 플랫폼 지원 (x86, ARM, PowerPC 등)
- **확장성**: 임베디드부터 슈퍼컴퓨터까지
- **보안**: 권한 분리, SELinux, AppArmor
- **네트워킹**: TCP/IP 스택 내장, 다양한 프로토콜 지원

### 4.2 커널 아키텍처
```
사용자 공간 (User Space)
├── 응용 프로그램
├── 시스템 라이브러리 (glibc)
└── 시스템 콜 인터페이스
─────────────────────────────
커널 공간 (Kernel Space)
├── 프로세스 관리
├── 메모리 관리
├── 파일 시스템
├── 네트워크 스택
└── 디바이스 드라이버
─────────────────────────────
하드웨어
```

**모놀리식 커널**:
- 모든 기능이 커널 공간에서 실행
- 높은 성능, 효율적인 통신
- 모듈 시스템으로 동적 로딩 지원

### 4.3 파일 시스템 철학
- **모든 것은 파일**: 디바이스, 프로세스 정보도 파일로 표현
- **단일 루트**: `/`에서 시작하는 통합 디렉토리 트리
- **마운트**: 다양한 파일 시스템을 트리에 연결
- **권한 모델**: 소유자, 그룹, 기타 사용자 구분

## 5. 리눅스의 활용 분야

### 5.1 서버 환경
- **웹 서버**: Apache, Nginx
- **데이터베이스**: MySQL, PostgreSQL, MongoDB
- **클라우드**: AWS, Google Cloud, Azure의 주요 OS
- **컨테이너**: Docker, Kubernetes
- **가상화**: KVM, Xen

### 5.2 임베디드 시스템
- **Android**: 리눅스 커널 기반 모바일 OS
- **IoT 디바이스**: 라즈베리 파이, 아두이노
- **네트워크 장비**: 라우터, 스위치 펌웨어
- **자동차**: 인포테인먼트 시스템

### 5.3 슈퍼컴퓨터
- **TOP500**: 상위 500대 슈퍼컴퓨터 중 대부분이 리눅스 사용
- **과학 연구**: 기상 예측, 유전체 분석, 물리 시뮬레이션
- **HPC (High Performance Computing)**: 병렬 처리 최적화

### 5.4 데스크톱 환경
- **개발자 워크스테이션**: 프로그래밍, 시스템 관리
- **교육 기관**: 비용 절감, 학습 목적
- **정부/공공기관**: 오픈소스 정책
- **데스크톱 환경**: GNOME, KDE, XFCE, Cinnamon

## 6. 리눅스 생태계

### 6.1 커뮤니티와 재단
- **Linux Foundation**: 리눅스 커널 개발 지원
- **CNCF**: 클라우드 네이티브 프로젝트
- **Apache Foundation**: 웹 서버 및 다양한 오픈소스 프로젝트
- **Mozilla Foundation**: Firefox, Thunderbird

### 6.2 상업적 지원
- **Red Hat**: IBM 인수, 엔터프라이즈 리눅스
- **Canonical**: Ubuntu 개발 및 지원
- **SUSE**: 유럽 중심 엔터프라이즈
- **Oracle**: Oracle Linux, 데이터베이스 통합

### 6.3 표준화
- **LSB (Linux Standard Base)**: 배포판 간 호환성
- **FHS (Filesystem Hierarchy Standard)**: 디렉토리 구조 표준
- **POSIX**: 유닉스 호환성 표준
- **systemd**: 초기화 시스템 사실상 표준

- **켄 톰슨, 데니스 리치**: UNIX 개발자

- **1969**: UNIX 탄생

- **1994**: Linux 1.0 릴리스

- **GNU/Linux**: 커널 + GNU 도구
