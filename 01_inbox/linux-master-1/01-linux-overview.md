# 리눅스 개요 및 역사 (심층 요약)

이 챕터는 리눅스 운영체제의 근간이 되는 철학, 역사, 라이선스, 그리고 현대 IT 기술(클라우드, 빅데이터 등)과의 연계를 다룹니다. 단순 암기보다는 개념의 차이를 이해하는 것이 중요합니다.

---

## 1. 운영체제(OS)의 이해

### 1.1 운영체제의 정의 및 목적
- **정의**: 하드웨어와 사용자 간의 매개체 역할을 하는 시스템 소프트웨어입니다.
    - **좁은 의미**: **커널(Kernel)** - 메모리에 상주하며 자원을 관리하는 핵심부.
    - **넓은 의미**: 커널 + **시스템 프로그램**(셸, 컴파일러, 유틸리티 등).
- **역할**: 하드웨어 제어, 자원 스케줄링(CPU, 메모리), 데이터 및 입출력 관리, 네트워크 처리, 오류 방지 및 UI 제공.

### 1.2 운영체제의 발달 과정 (★ 시험 포인트)

| 시대 | 주요 특징 | 관련 개념 |
| :--- | :--- | :--- |
| **1950년대** | 단순 순차 처리 | **Batch Processing** (일괄 처리) |
| **1960년대** | 다중 사용자/작업 시작 | Multi-Programming, **Time Sharing** (시분할) |
| **1970년대** | **UNIX 탄생** | C언어 기반 이식성, TCP/IP 표준화 |
| **1990년대** | 개방형 시스템 & GUI | Windows, **Linux 탄생** (1991), 분산 처리 |
| **2000년대~** | 고도화 & 확산 | 모바일, 임베디드, 클라우드 컴퓨팅, IoT |

### 1.3 주요 운영체제 종류
- **UNIX**: 1969년 켄 톰슨과 데니스 리치가 개발. C언어로 재작성되어 이식성이 높음.
    - **System V**: 상업용 중심 (IBM AIX, HP-UX, Solaris).
    - **BSD**: 학술 중심 (FreeBSD, **macOS의 기반**).
- **Windows**: MS-DOS에서 발전. GUI 기반, **PnP(Plug and Play)** 지원. Windows NT 커널 기반.
- **macOS**: 애플의 매킨토시용 OS. 마이크로커널(**Mach**)과 BSD 유닉스 기반(**Darwin**).

## 2. 리눅스 기초 및 철학

### 2.1 GNU 프로젝트와 FSF
- **GNU (GNU is Not Unix)**: 1983년 리처드 스톨먼이 시작. 유닉스 호환 자유 소프트웨어 개발 목적.
- **FSF (자유 소프트웨어 재단)**: 자유 소프트웨어 보급 지원 및 GPL 관리.
- **자유 소프트웨어의 4가지 자유**:
    1. 실행의 자유 (어떤 목적이든)
    2. 연구 및 수정의 자유 (소스 코드 접근 가능)
    3. 재배포의 자유 (이웃을 도움)
    4. 개작 및 배포의 자유 (공동체 발전)

### 2.2 오픈 소스 라이선스 비교 (★ 최다 빈출)

| 라이선스 | 소스 공개 의무 | 2차 저작물 공개 | 특징 |
| :--- | :--- | :--- | :--- |
| **GPL** | **있음** | **의무 (전연성)** | 가장 엄격함. 리눅스 커널, Git 등. |
| **LGPL** | **제한적** | 없음 (일부 조건) | 라이브러리 동적 링크 시 의무 완화. |
| **BSD** | **없음** | **없음** | 수정 후 비공개 상용화 가능. 매우 자유로움. |
| **Apache** | **없음** | **없음** | 아파치 재단 SW. 특허권 허용 조항 포함. |
| **MIT** | **없음** | **없음** | 가장 단순하고 제한이 적음. X Window 등. |

### 2.3 리눅스 커널 버전 표기법
- 예: `5.10.102`
    - **5 (주 버전)**: 커널의 획기적인 변화.
    - **10 (부 버전)**: 기능 추가 및 변경. (과거 2.6 이전엔 홀수 개발/짝수 안정이었으나 현재 구분 X)
    - **102 (패치 레벨)**: 버그 수정 및 보안 패치.

## 3. 리눅스 배포판 계열 (Family)

리눅스 커널에 GNU 소프트웨어와 애플리케이션을 결합한 형태입니다.

| 계열 | 특징 | 주요 배포판 | 패키지 도구 |
| :--- | :--- | :--- | :--- |
| **Red Hat** | 기업용 서버 시장의 강자. 상용화 및 기술 지원 중심. | RHEL, CentOS(무료 클론), Fedora(테스트베드) | **RPM, YUM, DNF** |
| **Debian** | GNU 정신을 가장 잘 계승. 높은 안정성과 전 세계적 커뮤니티. | Ubuntu(사용자 편의성), Debian, Kali(보안 테스팅) | **dpkg, APT** |
| **Slackware** | 가장 오래된 배포판 중 하나. 초기 설치/관리 난이도 높음. | Slackware, **SUSE**(슬랙웨어 기반에서 독립) | - (pkgtool) |

## 4. 리눅스의 기술적 특징

1. **계층적 파일 구조**: `/` (루트)를 기준으로 하는 **트리(Tree) 구조**. (Windows의 드라이브 개념과 다름)
2. **장치의 파일화**: 모든 장치(키보드, 디스크 등)를 파일처럼 취급 (**Everything is a File**).
    > [!TIP]
    > **명령어 vs 파일 기반 접근 (CPU 정보 확인 예시)**
    > - **lscpu**: CPU 아키텍처 정보를 인간이 읽기 좋게 요약해서 요약 출력해 주는 **시스템 명령어**입니다.
    > - **cat /proc/cpuinfo**: 커널이 관리하는 CPU 하드웨어 정보를 리얼타임으로 담고 있는 **파일**을 직접 읽는 방식입니다. '모든 것은 파일'이라는 리눅스 철학을 보여주는 대표적인 사례입니다.
    > - **기타 사례**: 메모리 정보(`free` 명령어 vs `/proc/meminfo`), 파티션 정보(`lsblk` 명령어 vs `/proc/partitions`).
3. **가상 메모리 (Swap)**: RAM 부족 시 하드 디스크 일부를 메모리처럼 사용.
4. **파이프(`|`) & 리다이렉션(`>`, `<`)**: 프로세스 간 통신 및 입출력 방향 전환.

## 5. 리눅스 활용 (클러스터링 & 클라우드)

- **클러스터링 (Clustering)**:
    - **HPC (고계산)**: 베어울프 클러스터. 병렬 연산으로 성능 극대화.
    - **LVS (부하분산)**: 로드 밸런서로 트래픽 분산.
    - **HA (고가용성)**: 장애 시 즉시 서비스 승계 (**무중단 서비스**).
- **클라우드 컴퓨팅 (Cloud)**:
    - **IaaS**: 인프라(서버, 스토리지) 임대 (AWS EC2).
    - **PaaS**: 개발 플랫폼 임대 (Google App Engine).
    - **SaaS**: 완성된 소프트웨어 서비스 (Google Docs, Dropbox).

## 6. 국가공인 리눅스마스터 실전 대비 문제 (10문항)

> [!NOTE]
> 정답은 각 문제 하단의 '해설' 부분을 확인하세요.

**문제 1. 다음 중 오픈 소스 라이선스에 대한 설명으로 올바르지 않은 것은?**
1. GPL은 프로그램을 실행, 연구, 수정할 자유를 보장하며, 수정된 프로그램을 배포할 경우 소스 코드를 반드시 공개해야 한다.
2. LGPL은 라이브러리를 동적으로 링크하여 사용하는 경우, 해당 응용 프로그램의 소스 코드를 공개할 의무가 없다.
3. BSD 라이선스는 2차 저작물의 소스 코드를 비공개로 하여 상용 소프트웨어로 배포하는 것을 허용한다.
4. Apache 라이선스는 소스 코드 수정 시 반드시 수정된 소스 코드를 원작자에게 제공해야 한다는 조항이 있다.
- **정답**: 4
- **해설**: Apache 라이선스는 소스 공개 의무가 없으며, 저작권 표시만 유지하면 됩니다. 수정 코드를 원작자에게 제공할 의무는 없습니다.

**문제 2. 리눅스 배포판과 패키지 관리 도구의 연결이 잘못된 것은?**
1. Debian - apt-get
2. CentOS - yum
3. Ubuntu - rpm
4. SUSE - zypper (YaST)
- **정답**: 3
- **해설**: Ubuntu는 데비안 계열이므로 dpkg/apt를 사용합니다. rpm은 레드햇 계열의 도구입니다.

**문제 3. 다음 설명에 해당하는 리눅스 클러스터링 기법은?**
*"두 대 이상의 서버를 연결하여 하나의 서버가 장애가 발생했을 때, 대기하고 있던 다른 서버가 즉시 서비스를 이어받아 서비스 중단을 방지하는 기술이다."*
1. HPC (High Performance Computing)
2. LVS (Linux Virtual Server)
3. HA (High Availability)
4. RAID (Redundant Array of Independent Disks)
- **정답**: 3
- **해설**: 서비스의 지속성을 보장하는 것은 HA(고가용성) 클러스터입니다.

**문제 4. 리눅스의 기술적 특징에 대한 설명 중 틀린 것은?**
1. 리눅스는 다중 사용자(Multi-user)와 다중 작업(Multi-tasking)을 지원한다.
2. 리눅스의 파일 시스템은 드라이브명(C:, D: 등)으로 구분되는 계층 구조를 가진다.
3. 가상 메모리(Swap)를 사용하여 물리적 메모리의 한계를 극복한다.
4. 파이프(Pipe) 기능을 통해 프로세스 간의 통신이 가능하다.
- **정답**: 2
- **해설**: 드라이브 문자는 윈도우의 특징입니다. 리눅스는 단일 루트(`/`) 기반의 트리 구조입니다.

**문제 5. 클라우드 컴퓨팅 서비스 모델 중, 소프트웨어 개발에 필요한 플랫폼(OS, 미들웨어, 런타임 등)을 가상화하여 제공하는 형태는?**
1. IaaS
2. PaaS
3. SaaS
4. DaaS
- **정답**: 2
- **해설**: 개발 플랫폼 환경을 제공하는 것은 PaaS(Platform as a Service)입니다.

**문제 6. 다음 중 GNU 프로젝트 및 FSF와 가장 거리가 먼 인물은?**
1. 리차드 스톨먼 (Richard Stallman)
2. 리누스 토발즈 (Linus Torvalds)
3. 켄 톰슨 (Ken Thompson)
4. 데니스 리치 (Dennis Ritchie)
- **정답**: 3, 4
- **해설**: 켄 톰슨과 데니스 리치는 UNIX의 원조 개발자로, GNU/FSF 프로젝트와는 직접적인 관계가 낮습니다. (리누스 토발즈는 커널을 GNU 시스템과 결합함)

**문제 7. 리눅스 커널 버전 5.8.12에서 각 숫자가 의미하는 바로 올바른 것은?**
1. 주 버전 - 패치 레벨 - 부 버전
2. 부 버전 - 주 버전 - 패치 레벨
3. 주 버전 - 부 버전 - 패치 레벨
4. 아키텍처 - 주 버전 - 부 버전
- **정답**: 3
- **해설**: Major(주) - Minor(부) - Patch Level(패치) 순서입니다.

**문제 8. 다음 중 모바일/임베디드 리눅스 운영체제에 대한 설명으로 틀린 것은?**
1. 안드로이드(Android)는 리눅스 커널 기반이며 자바/코틀린 언어를 주로 사용한다.
2. 타이젠(Tizen)은 리눅스 재단, 삼성전자, 인텔 등이 주도한 오픈 소스 모바일 OS이다.
3. 웹OS(WebOS)는 본래 팜(Palm)에서 개발했으나 현재는 LG전자가 스마트 가전에 활용 중이다.
4. iOS는 리눅스 커널을 기반으로 애플이 독자 개발한 모바일 운영체제이다.
- **정답**: 4
- **해설**: iOS는 리눅스가 아닌 BSD(Darwin/UNIX 계열) 기반입니다.

**문제 9. 리눅스 디렉터리 구조 중, 시스템 관리자가 사용하는 명령어(sbin)나 일반 사용자의 명령어(bin)가 위치하지 않는 디렉터리는?**
1. /bin
2. /usr/bin
3. /sbin
4. /etc
- **정답**: 4
- **해설**: `/etc`는 시스템 설정 파일(Configuration)이 위치하는 곳입니다.

**문제 10. 다음 중 리눅스에서 사용하는 셸(Shell)의 종류가 아닌 것은?**
1. bash
2. tcsh
3. zsh
4. command.com
- **정답**: 4
- **해설**: `command.com`은 MS-DOS 및 구버전 Windows용 명령어 해석기입니다.
