# visionOS Immersion Guide #apple #visionos #spatial

비전OS에서 몰입감 있는 앱을 만들기 위한 깊은 가이드. 용어는 [apple-glossary](../../00_foundations/apple-glossary.md).

## 공간 경험 설계
- 윈도우: 평면 UI, 컨트롤/리스트/폼에 적합.
- 볼륨: 3D 오브젝트/대화형 장난감/데이터 비주얼라이제이션.
- 풀 스페이스: 몰입형 경험, 패스스루/가상 환경 선택. 사용자 시야 전체를 차지하므로 안전/편안함 고려.

## 입력과 상호작용
- 시선: 기본 포인터. 강조/포커스를 자연스럽게 제공.
- 손 제스처: 핀치/드래그/회전/스크롤. 기본 제스처를 사용해 피드백 일관성을 유지.
- 음성: Siri/Dictation으로 텍스트 입력/명령. 키보드는 보조 수단.
- 주변 물리 공간: 앉은/선 자세, 테이블/바닥 레이아웃 고려.

## 렌더링/성능
- RealityKit/Metal로 3D/AR 콘텐츠. anchor, entity, component 구조 이해.
- 패스스루: 앱은 원본 카메라 프레임을 직접 받지 않고, 합성된 결과를 사용. 프라이버시·안전 필수.
- Foveated/Perceptual 렌더링으로 GPU 부하를 줄인다. 드로우콜/폴리곤/텍스처 최적화.
- 프레임 타깃: 90Hz 환경에서 안정적인 프레임 유지.

## 공간 오디오
- 객체 위치/거리/방향에 맞춘 소리. 사용자 머리 움직임에 따라 업데이트.
- 음향을 과도하게 사용하지 않고, 방향/거리 인지에 도움을 주도록 설계.

## UI 빌드 패턴
- SwiftUI + RealityView: UI와 3D 콘텐츠를 조합.
- 공간 레이아웃: 심도/거리/크기 일관성 유지. 너무 가까운 배치는 눈 피로 유발.
- 텍스트/컨트롤: 시스템 스타일 활용, 대비/투명도/빛 영향 확인.

## 안전/편안함
- 빠른 움직임/깜빡임/강한 대비는 줄인다.
- 이동은 텔레포트/페이드 등 부드러운 전환 사용. 강제 카메라 이동은 피한다.
- 사용자 시야 밖에서 갑자기 등장하는 요소를 피하고, 예측 가능한 동선을 제공.

## 데이터/프라이버시
- 공간 맵/메시/시선 데이터 접근은 제한적이며, 명확한 사용자 동의가 필요.
- 민감 데이터는 최소화하고, 저장/전송 시 암호화.

## 테스트
- 실제 기기/시뮬레이터 모두 확인. 시야/조명/거리/책상 높이 등 다양한 환경 테스트.
- 접근성: 음성/자막/콘트라스트/시선 민감도 설정.

## 배터리/열
- 3D/렌더링/센서가 많아 열/전력 소모가 크다. 작업을 분할하고 쉬는 시간을 둔다.
