# tvOS Media & Living Room Guide #apple #tvos #media

tvOS에서 거실용 앱/미디어 경험을 만들 때 필요한 내용을 쉽게 정리했다. 용어는 [[apple-glossary]].

## 거실 UX 원칙
- 멀리서 보는 큰 화면. 텍스트/버튼/포커스 상태를 큼직하게.
- 리모컨(Siri Remote) 중심 입력: 포커스 엔진, 스와이프/클릭/백/홈/음성.
- 짧은 입력, 긴 시청. 탐색/검색/시청 흐름을 단순하게.

## 포커스 엔진
- UIKit/SwiftUI에서 포커스 이동을 자동 처리. 커스텀 포커스 체인/우선순위를 설정할 수 있다.
- 포커스 변화 시 애니메이션/음향/스케일을 줘서 명확히 표시.

## 미디어 재생
- AVPlayer/HLS가 기본. 자막/다국어/오디오 트랙 지원.
- HDR/SDR/돌비 비전, 돌비 애트모스 지원 여부 확인. 기기/TV와 협상.
- Audio Session은 무음 스위치가 없으므로, 사용자 볼륨/자막 옵션 제공.

## 성능/전력
- 셋톱박스는 발열/전력이 비교적 자유롭지만, 과도한 GPU/CPU 사용은 스로틀링을 유발할 수 있다.
- 이미지/비디오 캐싱을 최적화. 큰 리소스는 온디맨드 리소스/서버 측 변환.

## 네트워크
- 와이파이/이더넷. 대역폭이 크지만, 품질이 나쁘면 버퍼링이 눈에 띈다.
- 퀵 재생을 위해 프리페치/로컬 캐시/적응형 비트레이트가 중요.

## 입력/보조 기능
- 음성 검색(Siri), 자막/오디오 설명, 색 대비/모션 감소 지원.
- 게임 컨트롤러/HID를 지원할 수 있다(게임/앱 종류에 따라).

## 앱 크기/온디맨드 리소스
- 초기 앱 크기 제한이 있다. 큰 자산은 On-Demand Resources(ODR)로 분리.
- 첫 실행 경험을 빠르게, 뒤에서 리소스 다운로드.

## 배포/정책
- tvOS App Store 심사: 비공개 API 금지, 콘텐츠 등급, 구매(IAP) 정책 준수.
- 백그라운드 실행은 제한적(음악/오디오 재생 등). 임의의 백그라운드 작업 불가.

## 테스트
- 다양한 TV/해상도/HDR 설정. 네트워크 느린 환경/와이파이 간섭.
- 리모컨 입력/포커스 상태/자막/다국어/이동 경로.
