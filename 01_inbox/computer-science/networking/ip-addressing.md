## 🌐 개요 (Overview)

**IP 주소 (Internet Protocol Address)** 는 [[tcp-ip-model|TCP/IP 네트워크]]에서 장치를 식별하는 고유한 주소입니다. 우편 주소처럼 네트워크에서 데이터가 올바른 목적지에 도달하도록 하는 역할을 합니다.

## 📍 IPv4 주소 체계

### 구조

- **32비트** (4바이트) 주소
- **점으로 구분된 10진수** (Dotted Decimal Notation)
- 예: `192.168.1.100`

```plaintext
192.168.1.100
 │   │   │  └─ 옥텟 4 (0-255)
 │   │   └──── 옥텟 3 (0-255)
 │   └────────옥텟 2 (0-255)
 └────────────옥텟 1 (0-255)
```

### 주소클래스 (Classful Addressing)

| 클래스 | 범위 | 기본 서브넷 마스크 | 네트워크/호스트 비트 | 용도 |
|--------|------|-------------------|---------------------|------|
| **A** | 1.0.0.0 ~ 126.255.255.255 | /8 (255.0.0.0) | 8/24 | 대규모 네트워크 |
| **B** | 128.0.0.0 ~ 191.255.255.255 | /16 (255.255.0.0) | 16/16 | 중규모 네트워크 |
| **C** | 192.0.0.0 ~ 223.255.255.255 | /24 (255.255.255.0) | 24/8 | 소규모 네트워크 |
| **D** | 224.0.0.0 ~ 239.255.255.255 | - | - | 멀티캐스트 |
| **E** | 240.0.0.0 ~ 255.255.255.255 | - | - | 예약됨 |

> [!NOTE]
> 현대 네트워크는 Classless (CIDR) 방식을 사용하므로 클래스는 레거시 개념입니다.

### 사설 IP 주소 (Private IP)

외부 인터넷에서 라우팅되지 않는 내부 네트워크용 주소:

| 범위 | CIDR | 설명 |
|------|------|------|
| 10.0.0.0 ~ 10.255.255.255 | 10.0.0.0/8 | 클래스 A 사설 |
| 172.16.0.0 ~ 172.31.255.255 | 172.16.0.0/12 | 클래스 B 사설 |
| 192.168.0.0 ~ 192.168.255.255 | 192.168.0.0/16 | 클래스 C 사설 |

**NAT (Network Address Translation)**:
- 사설 IP를 공인 IP로 변환
- 가정용 라우터에서 주로 사용

### 특수 IP 주소

| 주소 | 용도 |
|------|------|
| 127.0.0.1 | 루프백 (localhost) - 자기 자신 |
| 0.0.0.0 | 모든 인터페이스, 기본 라우트 |
| 255.255.255.255 | 브로드캐스트 (모든 호스트) |
| 169.254.0.0/16 | APIPA (자동 사설 IP) |

## 🎭 서브넷 마스크 (Subnet Mask)

네트워크 부분과 호스트 부분을 구분하는 역할:

```plaintext
IP 주소:       192.168.1.100
서브넷 마스크: 255.255.255.0
              ─────────── ───
              네트워크 부분 호스트 부분
```

### CIDR 표기법 (Classless Inter-Domain Routing)

`/숫자` 형식으로 네트워크 비트 수를 표현:

| CIDR | 서브넷 마스크 | 사용 가능 호스트 | 네트워크 개수 |
|------|--------------|-----------------|--------------|
| /8 | 255.0.0.0 | 16,777,214 | 1 |
| /16 | 255.255.0.0 | 65,534 | 256 |
| /24 | 255.255.255.0 | 254 | 65,536 |
| /30 | 255.255.255.252 | 2 | Point-to-Point 링크 |
| /32 | 255.255.255.255 | 1 (호스트 주소) | 단일 호스트 |

**계산 공식**:

```plaintext
사용 가능 호스트 수 = 2^(호스트 비트) - 2
                    (네트워크 주소 1개 + 브로드캐스트 주소 1개 제외)
```

**예시: 192.168.1.0/24**

```plaintext
네트워크 주소: 192.168.1.0
첫 호스트:     192.168.1.1
마지막 호스트: 192.168.1.254
브로드캐스트:  192.168.1.255
사용 가능: 254개
```

## 📐 서브네팅 (Subnetting)

큰 네트워크를 작은 서브넷으로 분할:

**예: 192.168.1.0/24를 4개 서브넷으로 분할**

| 서브넷 | 네트워크 주소 | 호스트 범위 | 브로드캐스트 | CIDR |
|--------|--------------|-------------|--------------|------|
| 1 | 192.168.1.0 | .1 ~ .62 | .63 | /26 |
| 2 | 192.168.1.64 | .65 ~ .126 | .127 | /26 |
| 3 | 192.168.1.128 | .129 ~ .190 | .191 | /26 |
| 4 | 192.168.1.192 | .193 ~ .254 | .255 | /26 |

**계산 방법**:
1. 필요한 서브넷 개수 → 2^n ≥ 필요 개수
2. 원래 CIDR + n = 새 CIDR (/24 + 2 = /26)
3. 블록 크기 = 256 / 4 = 64

## 🌈 IPv6 주소 체계

### 구조
- **128비트** (16바이트)
- **콜론으로 구분된 16진수** (8그룹)
- 예: `2001:0db8:85a3:0000:0000:8a2e:0370:7334`

### 축약 규칙

```plaintext
원본: 2001:0db8:0000:0000:0000:0000:0000:0001
축약: 2001:db8::1

규칙:
1. 앞쪽 0 생략: 0db8 → db8
2. 연속된 0000 ⇒ :: (한 번만 사용)
```

### 주소 타입

| 타입 | 범위 | 설명 |
|------|------|------|
| **Global Unicast** | 2000::/3 | 공인 IP (인터넷 라우팅 가능) |
| **Link-Local** | fe80::/10 | 로컬 네트워크만 (라우팅 불가) |
| **Unique Local** | fc00::/7 | 사설 IP (IPv4의 10.x.x.x와 유사) |
| **Multicast** | ff00::/8 | 그룹 통신 (브로드캐스트 대체) |
| **Loopback** | ::1/128 | 루프백 (IPv4의 127.0.0.1) |

### IPv4 vs IPv6 비교

| 특성 | IPv4 | IPv6 |
|------|------|------|
| **주소 길이** | 32비트 | 128비트 |
| **주소 개수** | 약 43억 개 | 340조×1조×1조 개 |
| **표기법** | 점-10진수 | 콜론-16진수 |
| **브로드캐스트** | 있음 | 없음 (멀티캐스트로 대체) |
| **NAT** | 필수 | 불필요 |
| **헤더** | 가변 길이 | 고정 길이 (더 빠름) |
| **자동 설정** | DHCP | SLAAC (무상태 자동 설정) |

## 💡 실무 활용 (Practical Usage)

### IP 주소 확인

```bash
# Linux
ip addr      # 권장
ifconfig     # 구식

# Windows
ipconfig

# macOS
ifconfig
```

### 서브넷 계산

```bash
# ipcalc 도구
ipcalc 192.168.1.0/24

# 출력:
# Address:   192.168.1.0
# Netmask:   255.255.255.0 = 24
# Network:   192.168.1.0/24
# Broadcast: 192.168.1.255
# HostMin:   192.168.1.1
# HostMax:   192.168.1.254
# Hosts/Net: 254
```

### 정적 IP 설정

```bash
# Linux (Debian/Ubuntu)
sudo ip addr add 192.168.1.100/24 dev eth0
sudo ip route add default via 192.168.1.1

# 영구 설정 (/etc/network/interfaces)
auto eth0
iface eth0 inet static
    address 192.168.1.100
    netmask 255.255.255.0
    gateway 192.168.1.1
```

### IPv6 확인

```bash
#IPv6 주소 보기
ip -6 addr

# IPv6 ping
ping6 2001:4860:4860::8888  # Google Public DNS
```

## 🏠 IoT와 IP 주소

**[[IPv6]] 는 IoT의 핵심**:
- 센서 하나하나에 공인 IP 할당 가능
- end-to-end 통신 (NAT 불필요)
- [[Thread]], [[Matter]] 프로토콜의 기반

**6LoWPAN**:
- IPv6를 IEEE 802.15.4 (저전력 무선)로 압축
- IoT 디바이스의 제한된 대역폭 해결
