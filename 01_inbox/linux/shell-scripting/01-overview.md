## 01. Bash 및 셸 기본 개념

리눅스 셸의 역할과 실행 환경, 그리고 명령어 해석의 기본 원리를 다룹니다.

### 1. 셸의 주요 역할

| 역할 | 설명 |
| :--- | :--- |
| **명령 인터프리터** | 사용자가 입력한 문자열을 파싱, 확장, 실행하는 역할을 수행합니다. |
| **스크립트 실행기** | 파일에 기록된 명령을 순차적으로 수행하며 변수, 루프 등을 제어합니다. |
| **환경 조율** | 변수, 함수, Alias, ulimit, 셸 옵션(`set`) 등을 관리합니다. |
| **잡 제어** | 프로세스의 포그라운드/백그라운드 전환 및 시그널 전달을 관리합니다. |

### 2. 셸 환경 설정 로드 순서 (Loading Order)

| 환경                    | 로드되는 파일 순서                                                                          |
| :-------------------- | :---------------------------------------------------------------------------------- |
| **인터랙티브 로그인 (SSH 등)** | `/etc/profile` → `~/.bash_profile` (또는 `~/.bash_login`, `~/.profile`) → `~/.bashrc` |
| **인터랙티브 비로그인 (터미널)**  | `~/.bashrc`                                                                         |
| **비인터랙티브 (스크립트 실행)**  | `$BASH_ENV` 변수에 지정된 파일 (지정 시에만)                                                     |

### 3. 명령어 해석 단계 (Execution Steps)

|  단계   | 내용                                                                   |                                                                      |
| :---: | :------------------------------------------------------------------- | -------------------------------------------------------------------- |
| **1** | **토큰화 (Tokenization)**: 공백을 기준으로 명령줄을 분리하고 인용 부호를 처리합니다.             |                                                                      |
| **2** | **확장 (Expansion)**: 브레이스 → 틸드 → 파라미터 → 명령 치환 → 산술 → 경로 확장 순으로 처리합니다. |                                                                      |
| **3** | **리디렉션 (Redirection)**: `<`, `>`, `>>` 등 입출력 방향을 설정합니다.              |                                                                      |
| **4** | **명령 찾기/실행**                                                         | **빌트인(Built-in)** → **함수(Function)** → **외부 명령(PATH)** 순으로 찾아 실행합니다. |

### 4. 주요 특수 문자 (Special Characters)

| 기호 | 설명 | 기호 | 설명 |
| :--- | :--- | :--- | :--- |
| **`*`** / **`?`** | 와일드카드 (0 개 이상 / 1 개) | **`\|`** | 파이프 (stdout → stdin) |
| **`&`** | 백그라운드 실행 | **`;`** | 순차 실행 (앞 명령 종료 후) |
| **`&&`** / **`\|\|`** | 논리 연산 (AND / OR) | **`#`** | 주석 |
| **`"`** / **`'`** | 인용 부호 (변수 해석 / 리터럴) | **`\`** | 이스케이프 (다음 문자 무력화) |
| **`` ` ``** / **`$()`** | 명령 대체 (실행 결과 반환) | **`$`** | 변수 참조 |
| **`()`** | 서브셸 실행 | **`{}`** | 브레이스 확장 / 명령 그룹화 |

### 5. 종료 상태 코드 (Exit Status)

|    코드     | 의미                    | 설명                                  |
| :-------: | :-------------------- | :---------------------------------- |
|   **0**   | **Success**           | 명령이 성공적으로 완료됨                       |
|   **1**   | **General Error**     | 일반적인 실패 (에러 발생 또는 미매칭)              |
|   **2**   | **Misuse**            | 셸 설정이나 옵션 사용 오류                     |
|  **126**  | **Permission Denied** | 명령은 찾았으나 실행 권한이 없음                  |
|  **127**  | **Command Not Found** | 명령어를 찾을 수 없음                        |
| **128+N** | **Signal Error**      | 시그널 N 에 의해 비정상 종료 (예: Ctrl+C = 130) |

### 5. POSIX sh vs Bash 주요 차약

| 기능        | POSIX sh (표준) | Bash (확장)                         |
| :-------- | :------------ | :-------------------------------- |
| **조건문**   | `[ ]` (test)  | `[[ ]]` (패턴/정규식 매칭 지원)            |
| **산술 연산** | `$(( ))`      | `(( ))`, `let` 등 확장 문법 지원         |
| **배열**    | 미지원           | 인덱스 배열, 연관 배열(4.0+) 지원            |
| **기타 기능** | 기본 리디렉션       | 프로세스 대체(`<( )`), 히어 스트링(`<<<`) 지원 |
