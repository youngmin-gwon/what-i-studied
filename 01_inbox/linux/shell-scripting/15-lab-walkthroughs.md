## 15. 실습 랩 워크스루

학습한 개념을 실제 환경에서 테스트하고 숙달하기 위한 단계별 가이드입니다.

### 1. 핵심 실습 랩 목록

| 랩 (Lab) | 학습 목표 | 핵심 단계 |
| :--- | :--- | :--- |
| **Lab 1: 인용 및 확장** | 단어 분리 및 인용 규칙의 차이 체감 | `set -x` 활성화 후 `$*`, `"$*"`, `"$@"` 출력 비교 |
| **Lab 2: 안전한 루프** | 공백/특수 문자가 섞인 파일 이름 처리 | `find -print0` 와 `read -d ''` 를 조합한 루프 작성 |
| **Lab 3: 파이프라인 에러** | 중간 단계 실패 감지 및 데이터 스트림 제어 | `set -o pipefail` 설정 및 `$PIPESTATUS` 배열 확인 |
| **Lab 4: 시그널 핸들링** | 스크립트 중단 시 안전한 자원 정리 | `trap` 을 사용한 임시 파일 자동 삭제 로직 구현 |
| **Lab 5: 옵션 파싱** | 커맨드라인 인자 및 옵션 처리 숙달 | `getopts` 를 활용한 `-f`, `-v` 등 옵션 처리 함수 작성 |

### 2. 랩 수행 및 검증 방법

| 단계 | 활동 (Activity) | 체크 포인트 |
| :---: | :--- | :--- |
| **1** | **환경 준비** | `mkdir`, `mktemp` 등을 사용하여 독립된 실습 공간 생성 |
| **2** | **코드 작성** | `set -Eeuo pipefail` 등의 안전 옵션을 포함하여 스크립트 작성 |
| **3** | **실행 및 관찰** | `bash -x` 를 통해 내부 동작 과정을 한 줄씩 모니터링 |
| **4** | **결과 검증** | `$?` (종료 코드)와 최종 출력 데이터의 일치 여부 확인 |

### 3. 확장 미션 (Advanced)

- **병렬 처리**: 동일 작업을 `xargs -P` 를 사용하여 4 개 이상의 코어에서 병렬 실행해보기.
- **포터빌리티**: 작성한 스크립트를 macOS(BSD)와 Linux(GNU) 양쪽에서 실행하며 `sed -i` 등의 차이점 수정하기.
- **최적화**: 루프 내에서 외부 명령 호출을 줄이고 Bash 내장 변수 확장을 사용하도록 리팩토링하기.
