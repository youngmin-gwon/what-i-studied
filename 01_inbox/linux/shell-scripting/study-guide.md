## 1 주 완성 셸 스크립트 학습 로드맵

단기간에 핵심을 파악하고 실전에 적용하기 위한 7 일 집중 학습 계획입니다.

### 1. 일자별 학습 계획

| 일차 (Day) | 학습 범위 (Files) | 중합 및 실습 과제 |
| :--- | :--- | :--- |
| **Day 1** | 01 ~ 03 | 인용/확장/변수 개념 파악, `shellcheck` 설치 및 기본 검사 |
| **Day 2** | 04 ~ 06 | 조건문/루프/리디렉션 집중, 파이프라인(`pipefail`) 실험 |
| **Day 3** | 07 ~ 08 | 텍스트 도구(sed, awk)와 시그널/잡 제어 실습, 로그 파이프라인 제작 |
| **Day 4** | 09 ~ 11 | 환경 변수/디버그/보안/포터빌리티 정리, Cron/Systemd 비교 |
| **Day 5** | 12 ~ 13 | 실전 레시피 적용 및 모의고사 풀기, 오답 노트(19 번) 작성 |
| **Day 6** | 14 ~ 16 | 명령 레퍼런스 탐독 및 트러블슈팅 쿡북 확인, 주요 랩(15 번) 실행 |
| **Day 7** | 17 ~ 21 | 셸 옵션/케이스/드릴 종합 복습, 나만의 표준 템플릿(18 번) 완성 |

### 2. 필수 점검 사항 (Top 5)

- [ ] **Quoting**: 모든 변수가 `"` 로 감싸져 있는가?
- [ ] **Error Path**: 실패 시 멈추거나 로그를 남기는가? (`set -e`, `trap`)
- [ ] **Portability**: Bash 전용 기능과 POSIX 표준을 구분할 수 있는가?
- [ ] **Input Safety**: 공백/특수 문자가 포함된 파일명을 안전하게 처리하는가?
- [ ] **Readability**: 타인이 이해하기 쉬운 스타일 가이드(10 번)를 따랐는가?
