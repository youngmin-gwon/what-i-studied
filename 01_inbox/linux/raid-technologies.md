## 🌐 개요 (Overview)

**RAID (Redundant Array of Independent Disks)** 는 여러 개의 하드 디스크를 하나의 논리적인 장치로 묶어 **성능 향상** 또는 **데이터 중복성(신뢰성) 보장**을 목적으로 사용하는 기술입니다.

---

## 🏗️ RAID 방식 (Hardware vs Software)

사용자는 특히 이 두 방식의 차이를 인지하는 것이 중요합니다.

| 구분 | 하드웨어 RAID (Hardware RAID) | 소프트웨어 RAID (Software RAID) |
| :--- | :--- | :--- |
| **제어 주체** | 전용 **RAID 컨트롤러 카드** (독립 CPU/Memory) | **운영체제 커널/CPU** (Linux의 `md` 드라이버) |
| **성능** | 높음 (시스템 CPU 부하 없음) | 보통 (시스템 CPU 자원 사용) |
| **비용** | 비쌈 (전용 카드 필요) | 저렴함 (추가 비용 없음) |
| **신뢰성** | 높음 (배터리 백업 캐시 등 활용) | 운영체제 상태에 의존적 |
| **관리 도구** | 컨트롤러 제조사 BIOS/유틸리티 | `mdadm` 명령어 |

## 📊 주요 RAID 레벨 (RAID Levels)

| 레벨 | 명칭 | 최소 디스크 | 핵심 특징 | 장점 | 단점 |
| :--- | :--- | :---: | :--- | :--- | :--- |
| **RAID 0** | Striping | 2 | 데이터를 나누어 분산 저장 | **속도 최상**, 용량 100% 사용 | 하나만 고장나도 **데이터 전멸** |
| **RAID 1** | Mirroring | 2 | 같은 데이터를 두 디스크에 복사 | **복구 능력 최상**, 안정성 | 용량 효율 50% (비용 높음) |
| **RAID 5** | Parity | 3 | 데이터와 **패리티(오류검사)** 정보를 분산 | 성능/안전성 균형 (효율적) | 디스크 2개 동시 고장 시 전멸 |
| **RAID 6** | Double Parity | 4 | 패리티를 서로 다른 디스크에 2중 저장 | 디스크 **2개 동시 고장**도 견딤 | 연산 부하가 높고 쓰기 성능 저하 |
| **RAID 10** | 1+0 | 4 | RAID 1(미러링)로 묶고 다시 RAID 0(스트라이핑) | 속도와 안정성 모두 확보 | 최소 디스크 4개 필요 (비용 높음) |

## 📝 자주 틀리는 문제 분석 (Exam Strategy)

사용자가 질문한 문제의 핵심 개념을 정리합니다.

> [!IMPORTANT]
> **RAID에 대한 오해와 진실**
> 1.  **스트라이핑 & 미러링**: RAID의 근간이 되는 기술이 맞습니다. (0=스트라이핑, 1=미러링)
> 2.  **데이터 무결성 검사**: RAID는 **패리티(Parity)** 또는 **ECC(Error Correcting Code)** 를 사용합니다. (참고: RAID 2는 ECC를 사용하지만 현재는 거의 퇴출되었고, RAID 3~6은 패리티를 사용합니다.)
> 3.  **용량 확장성 (⭐️ 정답 포인트)**: 일반적으로 RAID는 이미 구성된 상태에서 **"사용 중에 디스크 하나 꽂아서 바로 용량을 늘리는 것"** 이 기본 기능은 아닙니다. (LVM의 핵심 기능입니다.)
> 4.  **구현 방식**: 소프트웨어(OS 차원)와 하드웨어(전용 카드 차원)로 명확히 나뉩니다.

### RAID vs LVM 비교

| 특징 | RAID | LVM |
| :--- | :--- | :--- |
| **목적** | 데이터 안전성(중복성) 및 물리적 성능 | 파티션 관리의 **유연성** 및 확장성 |
| **온라인 확장** | 어려움 (재구성이 필요한 경우가 많음) | **매우 쉬움** (PV 추가 후 즉시 확장) |
| **결합 사용** | RAID로 안전한 디스크 풀을 만들고, 그 위에 LVM을 씌워 유연하게 사용함 (Best Practice) | |
