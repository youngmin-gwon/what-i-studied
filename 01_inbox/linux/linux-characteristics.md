## 🌐 개요 (Overview)

Linux는 Unix의 철학을 계승한 운영체제로, 초기 설계부터 다중 사용자, 다중 작업, 그리고 높은 이식성을 고려하여 개발되었습니다. 리눅스마스터 등 자격 시험뿐만 아니라 실무 이해를 위해서도 중요한 핵심 기술적 특징들을 정리합니다.

## 🔑 리눅스의 핵심 기술적 특징

### 1. 계층적 파일 구조 (Hierarchical File System)

리눅스와 유닉스는 모든 파일과 디렉토리를 **역트리(Inverted Tree) 구조**로 관리합니다.

*   **최상위 디렉토리(Root)**: **`/` (슬래시)** 가 리눅스의 최상위 디렉토리를 의미합니다.
*   **하위 디렉토리**: 모든 디렉토리는 루트(`/`) 아래에 계층적으로 존재합니다.

상세 내용은 [[filesystem-hierarchy-standard]]를 참조하세요.

> [!CAUTION]
> **자주 헷갈리는 기호**:
> *   **`/` (Root)**: 시스템의 시작점인 최상위 디렉토리입니다.
> *   **`.` (Current)**: 현재 내가 머물고 있는 디렉토리를 의미합니다.
> *   **`..` (Parent)**: 현재 디렉토리의 상위 디렉토리를 의미합니다.
> *   **`~` (Home)**: 현재 로그인한 사용자의 홈 디렉토리를 의미합니다.

### 2. 모든 장치의 파일화 (Everything is a File)

리눅스의 가장 중요한 철학 중 하나입니다. 하드웨어 장치조차 일반 파일과 동일한 인터페이스로 다룹니다.

*   **장치 파일**: 하드디스크(`/dev/sda`), 터미널(`/dev/tty`), 키보드, 네트워크 인터페이스 등이 `/dev` 디렉토리 아래 파일 형태로 존재합니다.
*   **장점**: 개발자는 복잡한 하드웨어 제어 레벨까지 몰라도 표준 파일 입출력(Open, Read, Write, Close) 함수를 통해 장치와 데이터를 주고받을 수 있습니다.

상세 내용은 [[file-types-links]]와 [[filesystems]]를 참조하세요.

### 3. 공유 라이브러리 사용 (Shared Libraries)

프로그램 실행 시 필요한 함수 뭉치(라이브러리)를 효율적으로 관리합니다.

*   **개념**: 여러 프로그램이 공통으로 사용하는 기능을 별도의 파일(`.so`, Shared Object)로 저장하고, 메모리에 **하나만 로드**하여 여러 프로세스가 **공유**합니다.
*   **장점**:
    *   **메모리 효율성**: 중복 로드를 방지하여 RAM 사용량을 줄입니다.
    *   **디스크 공간 절약**: 실행 파일 크기가 작아집니다.
    *   **업데이트 용이**: 라이브러리 파일만 교체하면 이를 사용하는 모든 프로그램에 즉시 반영됩니다.

### 4. 강력한 네트워크 및 입출력 제어

*   **리다이렉션 (Redirection)**: 기호(`<`, `>`, `>>`)를 사용하여 표준 입출력의 방향을 바꿀 수 있습니다.
*   **파이프 (Pipe)**: 기호(`|`)를 사용하여 한 명령어의 출력을 다른 명령어의 입력으로 즉시 전달합니다.

상세 내용은 [[shell-environment-commands]] 및 [[06-io-redirection]]을 참조하세요.
*   **네트워크 통합**: TCP/IP 프로토콜이 커널 수준에서 강력하게 구현되어 있어 서버 운영체제로서 매우 안정적입니다.

### 5. 다중 사용자 및 다중 작업 (Multi-User & Multi-Tasking)

*   **Multi-User**: 여러 사용자가 동시에 접속하여 각자의 환경에서 작업할 수 있습니다.
*   **Multi-Tasking**: CPU 스케줄링을 통해 여러 프로세스가 동시에 실행되는 것처럼 처리합니다. (시분할 방식 및 선점형 멀티태스킹)

## 📝 틀리기 쉬운 문제 분석 (Exam Strategy)

사용자가 틀린 문항을 기반으로 다시 확인해봅니다.

| 문항 | 분석 | 결과 |
| :--- | :--- | :--- |
| **1. 최상위 디렉터리인 .(root)를 기준으로 한다.** | 최상위(Root)는 **`/`** 입니다. **`.`**은 현재 디렉토리를 뜻합니다. | **틀림 (정답)** |
| **2. 모든 장치는 파일화해서 관리한다.** | "Everything is a File" 철학에 따라 맞습니다. | **옳음** |
| **3. 공유 라이브러리를 통해 메모리를 효율적으로 사용한다.** | `.so` 파일을 메모리에 한 번만 올려 공유하므로 맞습니다. | **옳음** |
| **4. 리다이렉션을 이용해서 입출력을 전환할 수 있다.** | `>`, `<` 등을 통해 입출력 방향 전환이 가능하므로 맞습니다. | **옳음** |

> [!TIP]
> 리눅스마스터 1급/2급 시험에서 시스템의 계층적 구조를 설명할 때 **`/` (Root Symbol)**을 **`.` (Current)** 혹은 다른 기호로 바꾸어 함정을 파는 경우가 매우 많습니다. 루트는 언제나 **슬래시(`/`)**임을 기억하세요!
