## 🌐 개요 (Overview)

리눅스 네트워크의 기초 개념: IP 주소 체계, 서브넷, 네트워크 설정 파일, 프로토콜 및 포트에 대해 다룹니다.

## 📍 IP 주소 클래스

### IPv4 클래스 구분

| 클래스 | 첫 번째 바이트 범위 | 기본 서브넷 마스크 | 용도 |
| :--- | :--- | :--- | :--- |
| **A** | 1 ~ 126 | 255.0.0.0 (/8) | 대규모 네트워크 |
| **B** | 128 ~ 191 | 255.255.0.0 (/16) | 중규모 네트워크 |
| **C** | 192 ~ 223 | 255.255.255.0 (/24) | 소규모 네트워크 |
| **D** | 224 ~ 239 | - | 멀티캐스트 |
| **E** | 240 ~ 255 | - | 예약 (실험용) |

>[!IMPORTANT]
>**시험 포인트**: 127.x.x.x 는 **루프백(Loopback)** 주소로 예약됨 (일반 목적 사용 불가)

### 사설 IP 대역 (Private IP)

| 클래스 | 사설 IP 범위 |
| :--- | :--- |
| A | 10.0.0.0 ~ 10.255.255.255 |
| **B** | **172.16.0.0 ~ 172.31.255.255** |
| C | 192.168.0.0 ~ 192.168.255.255 |

>[!TIP]
>**B 클래스 사설 IP**: 172.16.0.0 ~ 172.31.255.255 (172.15.x.x 나 172.32.x.x 는 포함되지 않음!)

### 루프백 주소

```bash
# 루프백 IP
127.0.0.1    # IPv4
::1          # IPv6

# 루프백 인터페이스 (리눅스)
lo           # 리눅스 표준 (lo0 아님!)
```

## 🔢 서브넷 계산

### 사용 가능한 호스트 수

```
기본 공식: 사용 가능한 호스트 = 2^(호스트 비트 수) - 2
(네트워크 주소와 브로드캐스트 주소 제외)
```

>[!IMPORTANT]
>**인터넷 연결 시**: 게이트웨이 주소도 제외해야 함!
> - 사용 가능 호스트 = 2^n - **3** (네트워크 + 브로드캐스트 + 게이트웨이)

| 서브넷 마스크 | 호스트 비트 | 이론상 | 인터넷 연결 시 |
| :--- | :--- | :--- | :--- |
| /24 (255.255.255.0) | 8 | 254 | **253** |
| /25 (255.255.255.128) | 7 | 126 | 125 |
| /26 (255.255.255.192) | 6 | 62 | 61 |
| /27 (255.255.255.224) | 5 | 30 | 29 |

### 서브네트워크 개수 계산

```
서브네트워크 개수 = 2^(빌린 비트 수)
```

**예시**: C 클래스 + 넷마스크 255.255.255.192

- 기본 C 클래스 마스크: 255.255.255.0 (/24)
- 설정된 마스크: 255.255.255.192 (/26)
- 빌린 비트: 26 - 24 = 2 비트
- 서브네트워크 개수: 2^2 = **4 개**

### 네트워크 주소 계산

**예시**: IP 192.168.3.129, 서브넷 255.255.255.128 (/25)

- 128 = /25 → 호스트 비트 7 개
- 네트워크 블록 크기: 128
- 129 > 128 → 네트워크 주소: **192.168.3.128**

## 📁 네트워크 설정 파일

### DNS 관련 파일

| 파일 | 설명 |
| :--- | :--- |
| `/etc/hosts` | 호스트명 -IP 매핑 (DNS 이전 방식, **DNS 의 효시**) |
| `/etc/resolv.conf` | DNS 서버 주소 설정 |
| `/etc/host.conf` | 호스트명 해석 순서 |
| `/etc/nsswitch.conf` | 이름 서비스 스위치 설정 |

```bash
# /etc/hosts
127.0.0.1   localhost
192.168.1.10  myserver.local

# /etc/resolv.conf (DNS 서버 확인)
nameserver 8.8.8.8
nameserver 8.8.4.4
search example.com
```

>[!IMPORTANT]
>**시험 포인트**: DNS 서버 확인 → `/etc/resolv.conf`
>(host.conf 아님!)

### 프로토콜 및 서비스 파일

| 파일 | 내용 |
| :--- | :--- |
| `/etc/protocols` | 프로토콜 이름 및 **프로토콜 번호** |
| `/etc/services` | 서비스 이름 및 **포트 번호** |

```bash
# /etc/protocols
tcp     6   TCP
udp    17   UDP
icmp    1   ICMP

# /etc/services
ssh            22/tcp
smtp           25/tcp
domain         53/tcp     # DNS
domain         53/udp     # DNS
http           80/tcp
https         443/tcp
```

## 🔌 프로토콜과 포트

### 주요 프로토콜 - 포트 매핑

| 프로토콜 | 포트 |
| :--- | :--- |
| **FTP** (데이터) | 20 |
| **FTP** (제어) | 21 |
| **SSH** | 22 |
| **Telnet** | 23 |
| **SMTP** | **25** |
| **DNS** | **53** |
| **HTTP** | 80 |
| **POP3** | 110 |
| **IMAP** | 143 |
| **HTTPS** | 443 |

>[!WARNING]
>**자주 틀리는 조합**: SMTP 는 53 이 아닌 **25**번 포트! (DNS 가 53)

### UDP 관련 프로토콜

**UDP**를 주로 사용하는 서비스:

- **DNS** (53) - 빠른 조회 필요
- **DHCP** (67, 68)
- **TFTP** (69)
- **SNMP** (161)

## 🌍 국제 표준 기구

| 기구 | 역할 |
| :--- | :--- |
| **ISO** | OSI 7 계층 모델 제정 |
| **IEEE** | LAN/MAN 표준 (802.3 이더넷 등) |
| **EIA** | RS-232C, UTP 케이블 규격 |
| **ITU-T** | 국제 전기 통신 표준화 |
| **ICANN** | **IP 주소 할당, 도메인 관리** |
| **IANA** | 포트 번호, 프로토콜 번호 관리 |

>[!TIP]
>**시험 포인트**:
> - OSI 모델 제정 → **ISO**
> - IP 주소/도메인 관리 → **ICANN**
> - LAN 표준 (802.x) → **IEEE**
> - 케이블 규격 (RS-232C) → **EIA**
