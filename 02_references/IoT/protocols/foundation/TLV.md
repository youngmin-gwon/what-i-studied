## 🌐 개요 (Overview)

**TLV (Type-Length-Value)** 는 데이터를 저장하거나 전송할 때 사용하는 매우 효율적인 **바이너리 인코딩** 방식입니다.
JSON 이나 XML 같은 텍스트 기반 형식에 비해 훨씬 적은 용량을 차지하며, 저성능 마이크로컨트롤러 (MCU) 에서도 빠르게 파싱할 수 있습니다.

>[!NOTE]
>[[Matter]] 는 모든 데이터 구조와 명령을 TLV 로 인코딩하여 전송합니다. 이는 배터리 및 메모리가 제한된 IoT 기기들에게 필수적인 선택이었습니다.

## 🏗️ 구조 (Structure)

데이터를 다음 세 부분으로 나누어 순서대로 나열합니다.

1. **Type (유형)**: 이 데이터가 무엇인지 (정수, 문자열, 불리언 등) 또는 어떤 태그인지 식별합니다. (보통 1 바이트)
2. **Length (길이)**: 이어지는 값 (Value) 의 크기 (바이트 수) 를 나타냅니다.
3. **Value (값)**: 실제 데이터입니다.

### 📝 예시 (개념적)

**"On"**이라는 상태를 보낸다고 가정해 봅시다.

- **JSON**: `{"status": "on"}` (약 15 바이트 소요)
- **TLV**: `0x01` (Bool Type) | `0x01` (Length 1) | `0x01` (Value True) (총 3 바이트)

## 🆚 JSON vs TLV 비교

| 특징        | **JSON / XML**       | **TLV (Matter)**         |
| :-------- | :------------------- | :----------------------- |
| **형식**    | 텍스트 (사람이 읽기 편함)      | **바이너리** (기계가 읽기 편함)     |
| **용량**    | 큼 (괄호, 따옴표 등 오버헤드 큼) | **매우 작음** (필요한 비트만 사용)   |
| **파싱 속도** | 느림 (문자열 해석 필요)       | **매우 빠름** (바이트 단위 처리)    |
| **디버깅**   | 쉬움 (눈으로 보임)          | 어려움 (별도 디코더 도구 필요)       |
| **사용처**   | 클라우드 API, 웹 개발       | **IoT 임베디드 통신**, 네트워크 패킷 |
